# Block Library — Versioning & Rollback Deep Dive — Demo Script

**Duration: 2:30**

**Goal:** Show that every AI prompt change is versioned, comparable, and reversible — so improving prompts is safe, not risky.

## Script

1. **Open with the concern.** *"Every records team I talk to has the same worry about AI — 'what happens when someone tweaks a prompt and makes it worse? How do we even know which version classified that document six months ago?' Let me show you how we remove that risk."*

2. **Navigate to AI → Blocks** and click the main classification prompt block.

3. **Show the Versions tab.** *"Every published change is an immutable version. You see version number, publisher, timestamp, and changelog message. This block has eleven versions going back six months — every one preserved, not one overwritten."*

4. **Open a version diff.** Click two versions to compare. *"Side-by-side diff. Additions in green, removals in red. Your compliance team sees exactly what changed between version 8 and version 9 — and why, because every publish has a changelog."*

5. **Demonstrate draft editing.** *"Edits happen in a draft — the live version keeps classifying documents. Your team can iterate safely. Preview the draft against a sample document before publishing."*

6. **Publish with a changelog.** *"When you publish, you must write a changelog — 'added example for contractor agreements based on correction feedback.' That entry is permanent."*

7. **One-click rollback.** *"If version 10 turns out to misclassify, click Rollback on version 9. The active version flips back, and a new version entry records the rollback — you never lose history."*

8. **Tie it to documents.** *"Every classified document records the exact block version that classified it. Your auditor asks 'what prompt classified this document?' — one click, you have the answer."*

**Key message:** *"AI prompt engineering stops being a black-box risk. Every change is traceable, comparable, and reversible — with the same rigour as code."*
