# Hub App — Pack Authoring & Publishing — Demo Script

**Duration: 3:00**

**Goal:** Show consultants, regulators, and industry bodies how to author and publish their own governance packs to the Hub — opening the platform into a marketplace.

## Script

1. **Open with the opportunity.** *"For consultancies, regulators, and industry associations — the Hub isn't just somewhere to consume packs, it's somewhere to publish them. If your firm has spent years refining a retention schedule for the legal sector, you can package it, publish it, and monetise it. Let me show you how."*

2. **Log into the Hub app** (`hub.*`). *"The Hub has its own admin surface, separate from tenant admin. This is where pack authors work."*

3. **Navigate to Packs → Create Pack.** *"Give the pack a name, a slug, a publisher, a licence. Pick the jurisdiction and the industry tags — these drive discoverability in the Hub listing."*

4. **Walk the pack editor tabs.** *"Authoring mirrors the component model you already know. A tab per component type — taxonomy, sensitivity, retention, metadata schemas, PII patterns, policies, traits, legislation references."*

5. **Build a taxonomy.** *"Drag and drop categories into a tree. Give each one a default sensitivity, a description, an ISO 15489 code if applicable. Link the legislation that drives that category — so when someone imports your pack, they can see 'this category exists because of UK Data Protection Act 2018 section 8'."*

6. **Show metadata schemas.** *"Define the structured fields the AI should extract for each category. Vendor name, matter reference, claim number — whatever is standard in your sector."*

7. **Show the PublishBar.** *"When you're ready, the PublishBar takes you through validation — missing fields, circular references, orphaned metadata schemas — and then lets you publish a new version. Every publish creates an immutable version history."*

8. **Show version management.** *"Once published, every tenant that imported your pack sees an update notification. You can publish hotfixes, deprecate old versions, and mark versions as recommended."*

9. **Mention licensing.** *"Packs can be free, paid, or subscription. The Hub handles entitlement automatically — a tenant only sees a pack's update if they have a valid licence."*

**Key message:** *"Turn your firm's domain expertise into a product. Publish once, every customer gets continuous updates — and you have a new recurring revenue stream."*
