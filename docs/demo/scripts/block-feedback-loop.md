# Block Library — Feedback Loop & AI Improvement — Demo Script

**Duration: 2:30**

**Goal:** Show how user corrections flow back to the specific prompt/pattern that produced them — and how admins can ask the AI to improve the block based on that feedback.

## Script

1. **Open with the idea.** *"Every correction your records manager makes is worth something — but only if it actually changes how the AI behaves next time. Otherwise you're doing the same manual work forever. Let me show you the closed loop."*

2. **Navigate to AI → Blocks** and open the classification prompt block.

3. **Show the Feedback tab.** *"Every time a reviewer overrides a classification, rejects a result, or flags missed PII, the feedback lands here — tied to the exact block version that produced the result."*

4. **Walk a correction entry.** *"A reviewer moved this document from 'HR > Miscellaneous' to 'HR > Leave Requests' — with the reason 'document is clearly a maternity leave confirmation'. We capture the original decision, the corrected decision, the reason, and a link back to the document."*

5. **Show feedback types.** *"Corrections, false positives (for PII), missed cases, and admin suggestions. Each is grouped by version so you can see whether the current version is accumulating more or fewer corrections than the last one — a signal of whether your prompt is improving or regressing."*

6. **Click "Improve with AI".** *"Once a block has collected enough feedback, the Improve with AI button lights up. Click it and the platform drafts a new version — incorporating the corrections as few-shot examples, tightening category descriptions, adding disambiguation guidance."*

7. **Review the draft.** *"The draft opens with a diff against the current version. Nothing is applied automatically — your prompt engineer reviews, edits if needed, and publishes. You stay in control."*

8. **Publish and watch.** *"Publish the new version. Next classification, the AI uses your improved prompt — and the feedback counter on the old version becomes your baseline for measuring the new one."*

**Key message:** *"Your reviewers' expertise compounds. Every override teaches the AI — and you watch the correction rate drop over time, with the evidence in plain sight."*
