# Hub App — Downloads & Version Management — Demo Script

**Duration: 2:00**

**Goal:** Show pack publishers how to see who's consuming their packs, at what version, and how to manage deprecations.

## Script

1. **Open with the publisher's perspective.** *"If you've published a pack to the Hub, you need to know — who's using it, what versions are in the wild, and when it's safe to deprecate. That's the Downloads view."*

2. **Navigate to Hub → Downloads** (`/downloads` in the Hub app).

3. **Show the overview.** *"For each of your published packs you see total installs, active tenants, version distribution — how many tenants are on 1.2 versus 1.3 versus 1.4."*

4. **Drill into a pack.** *"Click any pack to see the tenant-level breakdown. Tenant name, version installed, install date, last update check. If a tenant is three versions behind, it's obvious."*

5. **Show version-level stats.** *"For each published version — how many installs, how many updates away from it, how many conflicts tenants reported during update. That feedback informs your next release."*

6. **Demonstrate deprecation.** *"When you're ready to retire an old version, mark it deprecated. Tenants on that version get a heads-up notification in their update observer — 'this version will be unsupported from X date, please upgrade'."*

7. **Show the export.** *"Export any of this as CSV for your account management team — or pipe it via API into your CRM."*

**Key message:** *"You publish with confidence because you see the lifecycle of every pack across your entire customer base — and you can steer migrations with a nudge instead of a surprise."*
